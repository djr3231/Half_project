Toys API

HTTP Method Route Description Parameters / Request Body
GET /toys Returns A List Of Toys According To The query.
  • page: (optional): Page Number To Display (default: 1). For Example:
    page
  • perPage: (optional): Toys To Display Per Page (default: 5). For Example:
    perPage:
  • sort (optional): Query To Sort By (default: name, Descending Order). For Example:
    page:
  • reverse: (optional): If Reverse Is Set To yes, The Sort Will Be In Ascending Order, For Example:
    page
  • s: (optional): A search query by name or info field, returns a matching toy list to the query. For Example:
    page
  • category: (optional): The Name Of The Category That Will Return A Toy List Based On The Category. For Example:
    page
  • user_id: (optional): The user id associated with the toy returns a toy list associated with the user. For Example:
    page
GET /toys/count Returns a summary of the number of records and the number of pages per query Queries correspond to GET /toys: perPage, s, category & user_id.
POST /toys Adds a new toy
  • Request Body: Toy Data To Be Added Has To Follow The Format Below. 
    • 
  {
    "name": "String",
    "info": "String",
    "category": "String",
    "price": "Number"
  }
  • img_url: Attribute Is Allowed But Its Not Required.
  • user_id: Added automatically to Authenticated User.
  • date_created: Added automatically with execution time.
PUT /toys/:id Updates An Existing Toy.
  • :id : the ID of the toy to update.
  • Request Body: Toy Data To Be Added Has To Follow The Format Below. 
    • 
  {
    "name": "String",
    "info": "String",
    "category": "String",
    "price": "Number",
    "img_url": "String"
  }
  • img_url: Attribute Is Allowed But Its Not Required.
  • updatedAt: Added automatically with execution time.
DELETE toys/:id Deletes A Toy By Its ID. ID Of The Toy To Delete..

Users API

HTTP Method Route Description Parameters / Request Body
GET /users Test Endpoint To Check If The Server Is Up And Running. N/A
GET /users/userInfo Get The Iinformation For The Authenticated User (requires authentication). N/A
POST /users Create A New User.
  • Request Body: User to be added has to follow the format below: 
    • 
  {
    "name": "String",
    "email": "String",
    "password": "String"
    }
  • role: Attribute is not allowed, but created atomically as a "user".
POST /users/login Authenticate a user and receive a JWT token in response.
  • Request Body: User authentication should follow the format below: 
    • 
  {
    "email": "String",
    "password": "String"
  }
DELETE users/:id Deletes a user by its ID. ID Of the user to delete.
// admin Abillities

Admin vs User Abillities

Admin User Ability
V V Create a new toy or user record
V V Edit a self-created toy record.
V V Delete a self-created toy record.
V X Edit a toy record created by another user.
V X Delete a toy record created by another user.
V X Change the role of another user or admin
